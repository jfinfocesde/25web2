    ВїQuГ© es JSX?

    JSX (JavaScript XML) es una extensiГіn de sintaxis para JavaScript que permite escribir cГіdigo similar a HTML dentro de archivos JavaScript.     • Creado por: Facebook, como parte de React.
    • PropГіsito: Facilitar la creaciГіn de componentes de UI combinando lГіgica (JavaScript) y estructura (HTML-like) en un solo lugar.
    • No es HTML: JSX es una abstracciГіn que se compila a JavaScript puro, lo que lo hace mГЎs flexible y potente.

    ВїCГіmo funciona JSX?

    1. Escribir JSX:

    2. CompilaciГіn con Babel: JSX no es interpretado directamente por los navegadores. Herramientas como Babel (usadas automГЎticamente en Vite y Next.js) lo transforman en JavaScript:

    3. Renderizado: React toma estas llamadas a React.createElement para crear un DOM virtual (una representaciГіn ligera del DOM real). Luego, React compara el DOM virtual con el DOM real y actualiza solo las partes que cambiaron (reconciliaciГіn).

    4. IntegraciГіn en componentes: JSX se usa dentro de componentes React para definir su estructura. Por ejemplo:

  return <h1>Bienvenido a React</h1>;
}

    2. Expresiones JavaScript:

    3. Componentes como etiquetas:

    4. No es un template engine:

    5. Seguridad integrada:

    Reglas de JSX

    Para usar JSX correctamente, debes seguir estas reglas:

    1. Siempre cerrar etiquetas:

      • Todas las etiquetas deben cerrarse, incluso las vacГ­as: 
        <img src="image.jpg" alt="example" /> // Correcto
<img src="image.jpg"> // Incorrecto

    2. Un solo elemento raГ­z:

      • JSX debe devolver un solo elemento raГ­z. Si necesitas mГєltiples elementos, envuГ©lvelos en un <div> o un Fragment (<>): 
        return (
<>
    <h1>TГ­tulo</h1>
    <p>DescripciГіn</p>

    3. Usar className en lugar de class:

      • En JSX, los atributos HTML que coinciden con palabras reservadas de JavaScript se renombran: 
        <div className="container">Contenido</div>
      • Otros ejemplos: htmlFor en lugar de for, onClick en lugar de onclick.

    4. CamelCase para eventos y props:

      • Los eventos y propiedades personalizadas usan camelCase: 
        <button onClick={handleClick}>Clic</button>

    5. Expresiones en llaves {}:

      • Solo se permiten expresiones JavaScript (no bloques como if o for) dentro de {}: 
        <p>{2 + 2}</p> // Correcto

    6. Componentes comienzan con mayГєscula:

      • Los componentes personalizados deben empezar con mayГєscula para diferenciarlos de etiquetas HTML: 
        <MyComponent /> // Componente
